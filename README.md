# AO Rebirth

Preserving the legacy of Rubi-Ka. Building its future.

## Project Overview

AO Rebirth is an independent community project focused on researching,
preserving, documenting, and modernizing Anarchy Online server technology.

The project traces part of its technical heritage to CellAO and continues the
long-running community effort to understand Rubi-Ka's multiplayer systems,
gameplay infrastructure, and historical server architecture.

The current server workspace includes Login, Chat, Zone, and Web services, plus
shared libraries for data access, messaging, game entities, playfields,
inventory, combat, vendors, and related runtime systems. Historical stats and
enum documentation remain available in
[AORebirth/Documentation/Index.md](AORebirth/Documentation/Index.md).

## Mission Statement

AO Rebirth exists to preserve valuable community research, build a reliable and
maintainable server foundation, and support long-term exploration of the
systems, tools, and documentation needed to keep the technical legacy of
Rubi-Ka understandable for future developers.

## Core Principles

### Preservation

We preserve historical knowledge, prior community research, captured behavior,
technical notes, and implementation lessons so they are not lost.

### Modernization

We modernize architecture, tooling, validation, and development practices while
respecting the behavior and technical history that made the original work
valuable.

### Evolution

We treat AO Rebirth as a living project with its own roadmap: stable enough to
preserve the past, flexible enough to support future development.

## Project Goals

- Modern software architecture
- Reliable multiplayer infrastructure
- Comprehensive documentation
- Automated testing and validation
- Sustainable long-term development
- Community collaboration
- Historical preservation
